# claude-company

> Experimenting with Claude Code, I want to build a tool for myself to manage our division workforce without writing any code.

A web application for importing and viewing employee data from CSV files. The sample [employees.csv](employees.csv) was randomly generated by 
Claude and rolls up to the CEO "Peter Parker." 

Use cases (basic)

- A master CSV file determines the current division employees provided by our HR software

- Import CSV file - Allows to see what changed from the last import

- Team re-organization planning - Allows to drag-and-drop and move teammates around and export to CSV file


## Prerequisites

- Python 3.8+

```
sudo alternatives --set python /usr/bin/python3.12
sudo alternatives --set python3 /usr/bin/python3.12
```

## Setup

Create and activate a virtual environment, then install dependencies:

```bash
/usr/bin/python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

## Run

With the virtual environment activated:

```bash
source venv/bin/activate
python app.py
```

Then open **http://localhost:8080** in your browser.

## Deactivate

```bash
deactivate
```

## Container

### Prerequisites

- Podman / Docker

### Build

```bash
podman build -t company .
```

### Run

```bash
podman run --rm -p 8080:8080 company
```

Then open **http://localhost:8080** in your browser.

To persist employee data across container restarts, mount the data directory:

```bash
podman run --rm \
    --privileged \
    -p 8080:8080 \
    -v $(pwd)/data:/app/data company
```

## Usage

1. **Upload CSV** — select your employee CSV file and click **Import Employees**
2. **View Employees** — browse the imported records; expand the Attributes column for additional fields
3. Navigate to the other screens to see the organization workforce

## Notes

- The app replaces the employee store on each upload
- Employee data is saved to `data/employees.json` (created automatically on first import)
